Git init Estas en la carpeta en la que están los archivos de tu proyecto

abres la terminal y ejecutas git init para indicar ue esta en la carpeta central de mis archivos. Cuano ejecutamos el comando, además creamos:

  • Stating, es el area en la memoria RAM que es done el principio iremos agregando los cambios
  • Reository (repositorio/master): Es el directorio que conocemos como /.git/ que es el lugardonde al final del proyecto quedarán todos los cambios

    • git config - global user.name "mi nombre"

  • Configuramos el git a nuestro nombre

    • git config - global user.email "micorreo"

  • Configuramos el git a nuestro correo

    • git config --list

  • Nos muestra la configuración por defecta denustro git

    • git add archivo.txt

  • Agregamos el archivo al stating, en ete punto el archivo epera a que lo agreguemos al repositorio. Nota:antes de usar este comonano, el archivo lo llamamos como un archivo untrcked. Luego de usar el comando el archivo lo podemos llamar un archivo tracked, cuando está ya en el staging
  •   git add .Agregamos TOos los archivos dentro de nuestra carpeta Staging

    • git rm --cached archivo.txt

  • Eliminamos el archivo del staging area y lo volvemos Untracked

    • git commit -m "mensaje"

  • Luego de escribir un mensaje desriptivo de los cambios hechos, mandamos el archivo al repositorio/master. NOTA:Caa commit es una nueva versión con cambios de tu archivo

    • git status

  • Nos muestra la configuración por defecta denustro git

    • git log

  • Nos muestra el historial de commits hechos desde el mas reciente hasta el más viejo

    • git log --stat

  • Nos muestra el hilo de cambios especificos de cada commit desde el mas reciente hasta el mas viejo

    • git show archivo.txt

  • Muestra los cambios hechos al archivo en el último commit

    • git ditt

  • Nos muestra las diferencias entre los cambios que estan en staging y los que no se an añadido

    • git ditt #commit1 #commit2

  • Nos muestra las diferencias entre las dos versiones del commit

    • git reset #commit --solt

  • Volvemos nuestro archivo a la version del commit indicada, pero sin borrar los commits usados anteriormente

    • git reset #commit --hard

  • Volvemos nuestro archivo a la version del commit indicada, BORRANDO TODOS los commits usados anteriormente a la versión del commit indicada

    • git checkout #commit archivo.txt

  • Volvemos nuestro archivo a la versión del commit indicada, sin cambiar nada en el Staging
  •   git checkout master archivo.txt volvemos a nuestro archivo a la versión que teniamos antes de hacer el primer checkout, sin cambiar nada en el staging
